# RatePro GPT – Contract Review & Processing User Guide

## 1. Overview

The **RatePro GPT Contract Management Module** enables users to review, analyze, approve, or reject extracted contract data. The system organizes contracts into **batches** and provides filtering, searching, previewing, and approval workflows for efficient contract validation.

After logging into the RatePro GPT system, users are automatically directed to the **Extracted Contract Batches Landing Page**, where they can view and manage contract batches.

---

# 2. Landing Page – Extracted Contract Batches

When a user logs into **RatePro GPT**, they are redirected to the **Extracted Contract Batches page**.

This page displays all extracted contract batches and their current review status.

### Key Components

The landing page contains a **table** that displays the following information:

* Batch ID
* Supplier SAP Code
* Supplier Name
* Contract Number
* Contract Status
* Commodity
* Country
* Review Status (Reviewed / Approved / Rejected)

Each row represents a **batch of extracted contracts**.

---

# 3. Filtering Contract Batches

To help users locate specific contracts quickly, filtering options are available.

### Available Filters

Users can filter the contract batches using the following fields:

* Batch ID
* Supplier SAP Code
* Supplier Name
* Contract Number
* Contract Status
* Commodity
* Country

### How to Apply Filters

1. Select the desired filter field.
2. Enter the filter value.
3. The system will automatically filter and display the matching contract batches.

This helps users quickly locate specific batches or supplier contracts.

---

# 4. Downloading Batch Contract Data

Users can download contract data for a specific batch.

### Steps to Download Batch Data

1. Select the **Batch ID**.
2. Select the **Contract IDs** from the batch.
3. Once selected, the **Download button becomes enabled**.
4. Click the **Download** button.

### Output

The system generates and downloads an **Excel file** containing the contract details.

---

# 5. Opening Contract Extraction Page

Users can review extracted contract details from the landing page.

### Steps

1. Locate the required **Batch ID**.
2. Click the **Extraction Tab/Page Column**.
3. A **popup window** will appear.

This popup contains the list of **contracts associated with the selected batch**.

---

# 6. Contract Selection Popup

The popup displays a **table of contract IDs** belonging to the selected batch.

### Features

* Contract list table
* Search functionality
* Multi-selection option

### Search Option

A **search bar** is available above the table.

Users can search for:

* Contract ID
* Other related identifiers

The table will filter results based on the entered search criteria.

### Proceed to Contract Review

1. Select **one or more contracts**.
2. Click the **Proceed** button.

The user will then be redirected to the **Contract Details Page**.

---

# 7. Contract Details Page

The **Contract Page** provides a detailed view of the selected contract.

The page is divided into two main sections:

* Left Navigation Panel
* Right Contract Details Panel

---

# 8. Left Navigation Panel

The **left-hand navigation panel** contains the following:

### Search Functionality

Users can search contracts using:

* Contract ID

### Contract List

All contracts within the selected batch are displayed in the panel.

Users can:

* Click any contract ID
* Instantly view its details on the right-hand side.

---

# 9. Contract Details Panel

The **right-hand side panel** displays the detailed information of the selected contract.

Displayed information includes:

* Contract ID
* Extracted contract data
* Document preview options

---

# 10. PDF Preview

Users can view the original contract document.

### Steps

1. Click **Preview PDF**.
2. The PDF viewer opens on the **right side of the screen**.

### Additional Options

Users can also click:

**View Fullscreen**

This will open the full contract document in a **new browser tab** for better viewing.

---

# 11. Supplier Level Action

Users can perform actions at the **supplier level**.

### Steps

1. Click **Supplier Actions**.
2. A popup window called **Supplier Level Action** will appear.

### Available Options

Inside the popup:

* Supplier selection dropdown
* Approve option
* Reject option

### Process

1. Select the **Supplier** from the dropdown.
2. Choose either:

   * **Approve**
   * **Reject**
3. Click **Submit**.

The supplier-level action will be applied.

---

# 12. Extraction and Taxonomy Data View

Users can switch between two types of contract data:

* Extraction Data
* Taxonomy Data

### Switch Option

A toggle switch allows users to switch between:

* **Extraction View**
* **Taxonomy View**

### Extraction View

Displays the **raw extracted contract data** in a table.

### Taxonomy View

Displays the **structured taxonomy-based contract data**.

---

# 13. Download Contract Data

Users can download both extraction and taxonomy data.

### Steps

1. Click the **Download Button**.

### Output

The system downloads an **Excel file** containing:

Sheet 1: Extraction Data
Sheet 2: Taxonomy Data

---

# 14. Contract Approval and Reprocessing Options

Below the contract data table, users have **four workflow options**:

1. Approve C4U
2. Reprocess C4U
3. Approve Batch
4. Reprocess Batch

---

# 15. Approve C4U (Single Contract Approval)

This option approves the **selected contract**.

### Steps

1. Click **Approve C4U**.
2. A confirmation dialog appears:

"Are you sure you want to approve the following contract?"

3. Confirm the action.

The contract will be **approved**.

---

# 16. Reprocess C4U (Reject Contract)

This option rejects the **selected contract**.

### Steps

1. Click **Reprocess C4U**.
2. A popup confirmation dialog appears.

The user must select the rejection scope:

* Extraction
* Taxonomy
* Both

### Mandatory Reason

The user must enter a **mandatory rejection reason**.

After entering the reason:

1. Click **Reject C4U**

The contract will be **rejected**.

---

# 17. Approve Batch

This option approves **all contracts within the batch**.

### Steps

1. Click **Approve Batch**
2. Confirm the approval action.

All contracts in the selected batch will be approved.

---

# 18. Reprocess Batch

This option rejects **all contracts within the batch**.

### Steps

1. Click **Reprocess Batch**
2. Enter the **mandatory rejection reason**
3. Confirm the action.

All contracts in the batch will be rejected.

---

# 19. Status Tracking

Once a contract or batch is approved or rejected, the status is updated on the **Landing Page**.

The landing page displays:

* Contract Status
* Approved / Rejected Indicator
* User who performed the action
* Date and Time of the action

This allows users to track the **complete approval history** of each contract.

---

# 20. Summary

The RatePro GPT contract workflow allows users to:

* View extracted contract batches
* Filter and search contracts
* Review contract details
* Preview original contract documents
* Validate extraction and taxonomy data
* Approve or reject contracts
* Approve or reject entire batches
* Track approval history

This streamlined process ensures efficient **contract verification, validation, and approval workflows**.
